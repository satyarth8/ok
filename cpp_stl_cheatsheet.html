VECTOR
// #include <vector> — dynamic array, most used container
vector<T>
#include <vector>
Dynamic array — grows automatically. Random access O(1). Your go-to container.
WHEN TO USE

Need array that grows dynamically. Need random index access. Storing results to print later. Custom sort.

TRAPS

v[i] on empty vector = crash. push_back (not insert). pop_back removes LAST, not arbitrary.

ALL METHODS
push_back(x)Add x to endO(1) amort
pop_back()Remove last elementO(1)
v[i]Access index i (no bounds check)O(1)
v.at(i)Access with bounds check, throwsO(1)
v.front()First elementO(1)
v.back()Last elementO(1)
v.size()Number of elements (returns size_t)O(1)
v.empty()Returns true if size==0O(1)
v.clear()Remove all elementsO(n)
v.resize(n)Resize to n (fill with 0)O(n)
v.assign(n,x)Fill n elements with value xO(n)
v.insert(it,x)Insert before iteratorO(n)
v.erase(it)Erase at iteratorO(n)
v.begin()Iterator to firstO(1)
v.end()Iterator past lastO(1)
v.rbegin()Reverse iterator to lastO(1)
v.size() returns unsigned type. v.size()-1 when v is empty = HUGE number (underflow). Always check !v.empty() first.
Find frequency of each element and print in insertion order. Use vector to maintain order, map for count.
// ── Declaration ── vector<int> v; // empty vector<int> v(5); // size 5, all 0 vector<int> v(5, -1); // size 5, all -1 vector<int> v = {1,2,3,4}; // initializer list vector<vector<int>> mat(3, vector<int>(3,0)); // 3x3 matrix // ── Common ops ── v.push_back(10); v.push_back(20); cout << v[0]; // 10 cout << v.back(); // 20 v.pop_back(); // removes 20 // ── Iteration ── for(int x : v) cout << x << " "; // range-for for(int i=0; i<v.size(); i++) cout << v[i]; // ── Sort ── sort(v.begin(), v.end()); // ascending sort(v.begin(), v.end(), greater<int>()); // descending // ── Erase duplicates ── sort(v.begin(), v.end()); v.erase(unique(v.begin(), v.end()), v.end()); // sort first! // ── 2D vector input ── int n, m; cin >> n >> m; vector<vector<int>> mat(n, vector<int>(m)); for(int i=0;i<n;i++) for(int j=0;j<m;j++) cin >> mat[i][j];
push_back O(1) access O(1) insert/erase O(n) sort O(nlogn)

WHEN TO USE

Text processing, character manipulation, palindrome, anagram, pattern matching problems.

STACK
// #include <stack> — LIFO: Last In First Out
stack<T>
#include <stack>
LIFO structure. Top element accessible. No iteration. No index access. push/pop/top only.
WHEN TO USE

Bracket matching. Undo operations. Next greater element. Expression evaluation. DFS simulation.

TRAPS

stack.peek() DOES NOT EXIST in C++. Use top(). pop() returns void — get value with top() BEFORE pop(). No iteration.

ALL METHODS (only 5!)
st.push(x)Add element to topO(1)
st.pop()Remove top (returns VOID)O(1)
st.top()View top element (not removed)O(1)
st.size()Number of elementsO(1)
st.empty()True if no elementsO(1)
Java has stack.peek() — C++ has stack.top(). Also: ALWAYS check !st.empty() before calling top() or pop().
Balanced brackets: push open brackets, on close bracket check top() matches, then pop(). If stack empty at end → balanced.
// ── Balanced brackets ── string s = "({[]})"; stack<char> st; bool ok = true; for(char c : s){ if(c=='('||c=='{'||c=='[') st.push(c); else { if(st.empty()){ ok=false; break; } char top = st.top(); st.pop(); // get THEN pop if((c==')'&&top!='(')||(c=='}'&&top!='{')||(c==']'&&top!='[')) { ok=false; break; } } } if(!st.empty()) ok=false; cout << (ok ? "YES" : "NO"); // ── Next Greater Element ── vector<int> v = {4,5,2,25}; stack<int> stk; // stores indices vector<int> nge(v.size(), -1); for(int i=0; i<v.size(); i++){ while(!stk.empty() && v[stk.top()]<v[i]){ nge[stk.top()] = v[i]; stk.pop(); } stk.push(i); }
QUEUE / DEQUE
// #include <queue> and #include <deque>
queue<T> / deque<T>
#include <queue> / #include <deque>
Queue: FIFO. Deque: double-ended, push/pop from BOTH front and back.
WHEN TO USE

Queue: BFS, scheduling. Deque: sliding window max/min, when you need push/pop from both ends.

TRAPS

Queue: front() for first, back() for last. pop() removes FRONT (not back). No index access in queue.

QUEUE METHODS
q.push(x)Add to backO(1)
q.pop()Remove from FRONT (void)O(1)
q.front()First element (oldest)O(1)
q.back()Last element (newest)O(1)
q.size()Number of elementsO(1)
q.empty()True if emptyO(1)
DEQUE EXTRA METHODS
d.push_front(x)Add to frontO(1)
d.push_back(x)Add to backO(1)
d.pop_front()Remove from frontO(1)
d.pop_back()Remove from backO(1)
d[i]Random access by indexO(1)
d.front()/back()First/last elementO(1)
Queue pop() removes FRONT not back. front() = oldest element. back() = newest. Confused with stack? Stack: top() is newest. Queue: front() is oldest.
BFS level order traversal: push root, while queue not empty: pop front, process, push children.
// ── Queue basic ── queue<int> q; q.push(1); q.push(2); q.push(3); cout << q.front(); // 1 (oldest) cout << q.back(); // 3 (newest) q.pop(); // removes 1 cout << q.front(); // 2 now // ── Process queue ── while(!q.empty()){ int val = q.front(); q.pop(); cout << val << " "; } // ── Deque ── deque<int> d; d.push_back(1); d.push_front(0); // {0,1} d.push_back(2); // {0,1,2} d.pop_front(); // {1,2} cout << d[0]; // 1 — index works!
PRIORITY QUEUE
// #include <queue> — max-heap by default
priority_queue<T>
#include <queue>
Heap — always gives you MAX element at top. For min-heap, use greater<T> comparator.
WHEN TO USE

Kth largest/smallest element. Always need max or min efficiently. Dijkstra's algorithm. Job scheduling.

TRAPS

DEFAULT is MAX-heap (largest on top). For min-heap syntax is long: priority_queue<int,vector<int>,greater<int>>. pop() is void!

METHODS (same as stack!)
pq.push(x)Insert elementO(logn)
pq.pop()Remove top element (void)O(logn)
pq.top()View max (or min) elementO(1)
pq.size()Number of elementsO(1)
pq.empty()True if emptyO(1)
Min-heap declaration is verbose: priority_queue<int, vector<int>, greater<int>> pq; — don't forget vector<int> in the middle!
Find Kth largest: push all elements, pop (n-k) times, top() is answer. OR: maintain min-heap of size k.
// ── Max heap (default) ── priority_queue<int> maxpq; maxpq.push(3); maxpq.push(1); maxpq.push(5); cout << maxpq.top(); // 5 (MAX) maxpq.pop(); cout << maxpq.top(); // 3 // ── Min heap ── priority_queue<int, vector<int>, greater<int>> minpq; minpq.push(3); minpq.push(1); minpq.push(5); cout << minpq.top(); // 1 (MIN) // ── Kth largest element ── vector<int> v = {3,2,1,5,6,4}; int k = 2; priority_queue<int> pq; for(int x : v) pq.push(x); for(int i=0; i<k-1; i++) pq.pop(); cout << pq.top(); // 5 (2nd largest) // ── Priority queue of pairs (sorted by first) ── priority_queue<pair<int,int>> pqp; pqp.push({3,'a'}); pqp.push({1,'b'}); cout << pqp.top().first; // 3
push O(logn) pop O(logn) top O(1)

UNORDERED MAP/SET
// #include <unordered_map> / #include <unordered_set> — O(1) average
unordered_map / unordered_set
#include <unordered_map> / <unordered_set>
Hash table — O(1) average insert/find. NOT sorted. Faster than map/set for large inputs.
WHEN TO USE

When you need map/set but DON'T need sorted order. Faster O(1) vs O(logn) for large n. Freq count where order doesn't matter.

TRAPS

Worst case O(n) per op (hash collision). No ordered iteration. lower_bound/upper_bound NOT available. Output order unpredictable.
